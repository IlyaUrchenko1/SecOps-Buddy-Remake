# MITRE ATT&CK T1030

## Официальный источник

- ATT&CK техника: [T1030 — Data Transfer Size Limits](https://attack.mitre.org/techniques/T1030/)

## Что означает T1030

Техника описывает ситуацию, когда злоумышленник намеренно ограничивает размер передаваемых блоков данных, чтобы скрыть эксфильтрацию и снизить вероятность срабатывания DLP/IDS/аномалий по объему трафика.

Обычно это выглядит как серия небольших передач вместо одного большого выгружаемого потока.

## Как это отражено в MVP

Текущая реализация в проекте не делает packet-level измерения размеров блоков. Вместо этого используется поведенческая эвристика:

- множественные snapshot-ы сетевых соединений
- группировка по `(remote_ip, remote_port, proto, process_name)`
- оценка повторяемости соединений
- оценка разнообразия локальных ephemeral портов
- штрафы для private/local/safe-сценариев

## Почему это честный MVP

`ss`/`netstat` дают хорошую оперативную картину сокетов, но не дают точной семантики размера «чанков» передачи. Поэтому текущий детектор — это proxy-подход для раннего обнаружения подозрительного паттерна.

## Как улучшать дальше

- добавить collector на eBPF или pcap
- анализировать байты/пакеты по flow
- учитывать базовые профили нормального поведения хостов
- добавлять корреляцию с процессами, DNS, историей репутации IP
